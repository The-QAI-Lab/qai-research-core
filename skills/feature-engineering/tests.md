# Feature Engineering Skill — Behavioral Validation Tests

- Are all features relevant, justified, and assigned to explicit families (raw, derived, domain-informed, exploratory)?
- Are calculations, value ranges, rationale, and hypothesis/analysis linkage explicit for each feature?
- Are leakage risks identified and mitigation strategies proposed?
- Are validation and ablation suggestions present?
- Is the Feature Handoff Block present and machine-readable?
- Is the output structured with explicit section headers and rationale for feature selection?
- Does the skill avoid generic, unjustified, or non-reproducible features?
- Are domain-specific needs (e.g., normalization, harmonization) addressed?
- Is the output suitable for direct use by Statistical Analysis and Paper Outliner skills?
- Does the skill request clarification if dataset/schema or research goal is ambiguous?
- Does it handle errors and missing data gracefully, reporting them explicitly?
