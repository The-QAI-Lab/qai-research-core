# Protocol Writer Skill — Behavioral Validation Tests

- Are all protocol steps explicit, reproducible, and justified, with step-to-design rationale mapping?
- Are operational failure points, checkpoints, and deviations tracking surfaced?
- Is the materials list complete, with quantities and specifications?
- Does the protocol address compliance (IRB, reporting standards, operational constraints) with a checklist?
- Are documentation requirements and handoff blocks present and structured?
- Is the output structured with explicit section headers and rationale for steps/materials?
- Does the skill avoid vague, incomplete, or unjustified instructions?
- Is the output suitable for direct use by Dataset Schema Designer and Dataset QA skills?
- Does the skill request clarification if the experiment plan or materials are ambiguous?
- Does it handle errors and missing data gracefully, reporting them explicitly?
