# Gap Finder Skill — Behavioral Validation Tests

- Does the skill extract both explicit and implicit gaps from structured literature review outputs?
- Are all gaps actionable, specific, and non-generic?
- Is each gap traceable to a section or citation in the literature review?
- Is the output organized with explicit section headers (e.g., Gap Prioritization Matrix, Experiment Framing Block)?
- Does the skill avoid restating known findings as gaps?
- Does the output include a prioritization matrix and candidate research questions?
- Does the output include an Experiment Framing Block suitable for direct use by Experiment Design and Paper Outliner?
- Is the output format compliant with `standards/output-contracts.md`?
- Does the skill handle ambiguous or missing input gracefully, reporting errors explicitly?
- Does the skill surface assumptions, uncertainties, and tradeoffs in gap prioritization?
- Does the skill distinguish evidence-backed gaps from speculative or inferred gaps?
