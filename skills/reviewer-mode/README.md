# Reviewer Mode Skill

A high-rigor reasoning module for emulating standards-driven peer review of manuscripts, protocols, or research outputs. This skill surfaces fatal flaws, fixable weaknesses, and provides prioritized, actionable revision guidance. Outputs are designed for internal QA and pre-submission review, with explicit Reviewer Exposure Block integration from Paper Outliner.

- **Purpose:** Deliver structured review reports, actionable feedback, rubric scores, and revision priority ranking, with explicit fatal flaw/weakness distinction and Reviewer Exposure Block integration.
- **When to Use:** After manuscript or protocol drafting, before submission or publication, or for internal QA.
- **When Not to Use:** If input document or review rubric is missing, ambiguous, or unreviewed.
- **Related Standards/Assets:** `shared/review-rubric.md`, `standards/evaluation-checklist.md`, `skills/paper-outliner/skill.md`, `standards/output-contracts.md`
- **Upstream:** Paper Outliner (Reviewer Exposure Block)
- **Downstream:** (End of pipeline)
- **Handoff Artifact:** Reviewer Exposure Block (machine-readable: weak sections, missing evidence, likely reviewer concerns)
- **Reasoning Features:** Fatal flaw/weakness distinction, prioritized revision guidance, rubric integration, Reviewer Exposure Block handling, explicit standards referencing
