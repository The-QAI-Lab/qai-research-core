# Reviewer Mode Skill — Behavioral Validation Tests

- Is feedback specific, actionable, and standards-compliant?
- Are all rubric criteria addressed, justified, and referenced in the report?
- Are fatal flaws and fixable weaknesses explicitly distinguished and prioritized?
- Is the Reviewer Exposure Block integrated and machine-readable?
- Is the revision priority ranking present and actionable?
- Is the review report structured with explicit section headers and rationale for each point?
- Does the skill avoid missing critical issues or providing generic feedback?
- Are domain-specific needs (e.g., sensor calibration, harmonization) addressed?
- Is the output suitable for direct use by authors and internal QA?
- Does the skill request clarification if input document or rubric is ambiguous?
- Does it handle errors and missing data gracefully, reporting them explicitly?
