# Dataset Schema Designer Skill — Behavioral Validation Tests

- Are all fields, types, value ranges, and validation rules explicitly defined and justified?
- Are metadata, provenance, and versioning explicit and standards-driven?
- Are schema risks and harmonization considerations surfaced?
- Are missing data policies included and explicit?
- Are downstream analysis implications discussed?
- Is the QA Validation Block present and machine-readable?
- Is the output structured with explicit section headers and rationale for field choices?
- Does the skill avoid ambiguous, incomplete, or unjustified definitions?
- Is the output suitable for direct use by Dataset QA and Feature Engineering skills?
- Does the skill request clarification if research goals or data collection plan are ambiguous?
- Does it handle errors and missing data gracefully, reporting them explicitly?
