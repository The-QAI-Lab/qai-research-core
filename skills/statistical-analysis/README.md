# Statistical Analysis Skill

A high-rigor reasoning module for constructing, justifying, and documenting explicit, standards-driven statistical analysis workflows. This skill connects hypotheses to tests, surfaces all assumptions and uncertainties, analyzes tradeoffs, and produces structured, machine-readable outputs for downstream research skills.

- **Purpose:** Deliver auditable analysis plans, statistical test tables, and reporting structures with explicit assumption, uncertainty, and tradeoff handling. Outputs are designed for direct use by Paper Outliner and Reviewer Mode.
- **When to Use:** After feature engineering and Feature Handoff Block are available, before or during data analysis.
- **When Not to Use:** If dataset or feature list is missing, ambiguous, or unreviewed.
- **Related Standards/Assets:** `skills/feature-engineering/skill.md`, `skills/paper-outliner/skill.md`, `standards/output-contracts.md`
- **Upstream:** Feature Engineering (Feature Handoff Block)
- **Downstream:** Paper Outliner, Reviewer Mode
- **Handoff Artifact:** Paper Claims Block (machine-readable: claims, evidence, limitations, open questions)
- **Reasoning Features:** Hypothesis-to-test linkage, explicit assumption and uncertainty reporting, tradeoff analysis, primary/fallback analysis, power/sample size concerns, interpretation cautions, exploratory vs. confirmatory distinction, structured handoff for downstream skills
