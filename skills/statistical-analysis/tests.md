# Statistical Analysis Skill — Behavioral Validation Tests

- Are all analysis steps explicit, justified, and standards-compliant?
- Is hypothesis-to-test linkage explicit for each analysis?
- Are assumptions checked and documented for each method?
- Are primary and fallback analysis paths present?
- Are power/sample size and interpretation cautions surfaced?
- Is the exploratory vs. confirmatory distinction explicit?
- Is the Paper Claims Block present and machine-readable?
- Is the output structured with explicit section headers and rationale for each step?
- Does the skill avoid generic, unjustified, or non-reproducible analysis plans?
- Are domain-specific needs (e.g., repeated measures, harmonization) addressed?
- Is the output suitable for direct use by Paper Outliner and Reviewer Mode skills?
- Does the skill request clarification if dataset/features or hypotheses are ambiguous?
- Does it handle errors and missing data gracefully, reporting them explicitly?
