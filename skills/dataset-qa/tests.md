# Dataset QA Skill — Behavioral Validation Tests

- Does the skill identify all major data issues, including completeness, consistency, standards compliance, and downstream risk?
- Are issues classified by severity and impact (blocking/non-blocking)?
- Are root causes and remediation priorities surfaced?
- Are recommendations specific, actionable, and prioritized?
- Is the Feature Risk Block present and machine-readable?
- Is the QA report structured with explicit section headers and rationale for findings?
- Does the skill avoid generic, incomplete, or unjustified checks?
- Are domain-specific criteria (e.g., sensor calibration, harmonization) addressed?
- Is the output suitable for direct use by Feature Engineering and Statistical Analysis skills?
- Does the skill request clarification if schema or dataset sample is ambiguous?
- Does it handle errors and missing data gracefully, reporting them explicitly?
